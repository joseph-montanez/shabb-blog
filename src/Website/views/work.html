Relative Deployment

Relative Deployment is a Microsoft Windows process controller ideally for running web applications on the desktop. The key feature is finding available ports to deploy a process such as a webserver, database or other process. you can open a web address to the users default browser, or use something like Mozilla's XulRunner to a controlled browser UI experience. The application is tiny, uses JSON for configurations and is designed for a stub executable for you to adopt as your application. Try downloading the source at Github.

Shado List

Shado List is a shared to do list built on similar techniques that NodeJS uses, but written in PHP using a library called React. This proof of concept uses React and jQuery Mobile together, bringing a cohesive experience to the end user. Shado uses websockets to deliver real-time communications between multiple users of the same shared list. Visit the demo application, Shado List. Wazillo Media

Wazillo Media is a website that streams and records live content. My role in this website was to integrate Wowza Streaming Server and Flowplayer. The content streamed live needed to be recorded on the same server so I developed a Java plugin for Wowza. Also all recordings needed to be automatic which relied on a cron job. There is another side to this, the company provides streaming services for bars and venues. I was in charge of integrating rtsp streams into Wowza from each venue.


Inner Greek

Inner Greek is a social fraternity and sorority platform. My roles in this application was to create a real-time notification system, for incoming requests and messages. I also created this application's billing system for handling subscriptions. Real-time communications used the same technology that I had established. NodeJS enabled myself to create a web server that allowed thousands of connections per second. Socket.IO is a web socket library that falls back to comet and flash tcp messages. NodeJS and Socket.IO provided the starting point of building the real-time system. There are other factors into building a real time system, which relied on trusted messages. Since the logic lived in PHP I had to build a transport library for PHP to have the ability to communicate to NodeJS. Also things like multi-tab browsers created some complications to work around. I needed to ensure all browsers and tabs receive the same message of that specific user, and marking a notice "read" deactivates all others in the other browsers and tabs.


San Diego Bike and Kayak

San Diego Bike and Kayak is a tour and rental system. It features a customer, in-store and concierge portal. The in-store provides real-time information to allow staff to see what rentals and tours are available or have the least capacity. Coupled with vouchers, discounts and coupons, the staff has all the controls needed to setup and price given the the reduction options. Each tour has a multi-tier attribute structure. An attribute can be price there is a general, grouped, daily and a to the hour price. There are certain limitations at specific times due to city regulation. For example X persons or Y kayaks on the water at a given time, which ever is met first.


Sitbetter

Sitbetter is a shopping site for chairs and accessories. It features a powerful drill down engine, built with speed and ease of use. Chairs have extensive configurations available such as material selector. The drill down menu creates over 500,000,000+ different combinations of chairs, which created a performance issue. A caching system was set in place to quickly generating any drill down page. This entire system was built from scratch and features amazon integration, tons of exports including shopzilla, google shopping and more.
